# Review Checklist

Use this before approving UI work.

## Product

- Who is the user?
- What is the user trying to accomplish?
- What is the fastest path to success?
- Is the primary action obvious?
- Is the next step clear?

## Simplicity

- Can anything be removed?
- Is every element necessary?
- Is complexity progressively disclosed?
- Are rare actions visually secondary?

## Information Architecture

- Is essential information visible?
- Is important information secondary?
- Is advanced information hidden until needed?
- Does the layout match the user's mental model?

## Risk

- What is the cost of mistakes?
- Are high-risk actions protected?
- Is destructive action visually secondary?
- Is undo or recovery available where appropriate?

## Accessibility

- Is keyboard navigation supported?
- Are focus states visible?
- Are labels and headings correct?
- Can screen readers understand the UI?

## States

- Loading state exists.
- Empty state exists.
- Error state exists.
- Success state exists.

## Final Question

Would a first-time user understand this screen within five seconds?
